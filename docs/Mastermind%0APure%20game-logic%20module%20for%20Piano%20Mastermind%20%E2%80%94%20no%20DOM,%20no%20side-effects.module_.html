(inner) get_feedback(secret, guess) → {Feedback}

Description:

  • Score a single guess against the secret code.

    A green peg means the note is correct and in the right position. A red peg means the note appears in the secret but in a different position. Each note in the secret can only account for one peg.

Source:
  • Mastermind.js, line 75
Example
get_feedback(["C","D","E","F"], ["C","E","D","F"])
// => { greens: 2, reds: 2 }
Parameters:
Name Type Description
secret Secret

The hidden code chosen by the computer.
guess Guess

The player's attempt, same length as secret.
Returns:

Object with greens and reds counts.

(inner) get_positional_feedback(secret, guess) → {Array.<PegColour>}

Description:

  • Return per-position feedback for a guess — "green", "red", or "empty" for each slot.

    Unlike get_feedback which returns counts, this maps each position directly to a colour so the UI can label individual pegs. Green means right note, right position; red means right note, wrong position; empty means that note does not appear in the secret at all.

Source:
  • Mastermind.js, line 117
Example
get_positional_feedback(["C","D","E","F"], ["C","F","E","G"])
// => ["green", "red", "green", "empty"]
