(inner) get_feedback(secret, guess) → {Feedback}

Description:

  • Score a single guess against the secret code.

    A green peg means the note is correct and in the right position. A red peg means the note appears in the secret but in a different position. Each note in the secret can only account for one peg.

    Curried - partially apply the secret to get a reusable scorer: const score = get_feedback(secret); score(guess1); score(guess2);

Source:
  • Mastermind.js, line 82
Example
get_feedback(["C","D","E","F"], ["C","E","D","F"])
// => { greens: 2, reds: 2 }
Parameters:
Name Type Description
secret Secret

The hidden code chosen by the computer.
guess Guess

The player's attempt, same length as secret.
Returns:

Object with greens and reds counts.

(inner) get_positional_feedback(secret, guess) → {Array.<PegColour>}

Description:

  • Return per-position feedback for a guess - "green", "red", or "empty" for each slot.

    Unlike get_feedback which returns counts, this maps each position directly to a colour so the UI can label individual pegs. Green means right note, right position; red means right note, wrong position; empty means that note does not appear in the secret at all. Each note in the secret can only account for one peg - a note cannot score both a green and a red simultaneously.

    Curried - partially apply the secret: const score_pos = get_positional_feedback(secret);

Source:
  • Mastermind.js, line 126
Example
get_positional_feedback(["C","D","E","F"], ["C","F","E","G"])
// => ["green", "red", "green", "empty"]
Parameters:

(inner) make_guess(guess, game) → {Game}

Description:

  • Return a new game state with the guess (and its feedback) appended. The original game state is never modified - each call returns a fresh game object with the new attempt added.

    If the game is already over (won or lost), the state is returned unchanged - extra guesses are silently ignored.

    Curried - partially apply the guess: const try_c = make_guess(["C","C","C","C"]);

Source:

// => game with 1 attempt, feedback ["green","empty","empty","empty"]

(inner) random_chord_secret(rng, count, pool) → {Array.<Note>}

Description:

  • Sample count unique notes from pool without replacement. No note appears twice in the result - each selected note is excluded from the draw for subsequent picks. Each call with a seeded rng produces the same result (pure, deterministic).

    Curried - partially apply rng + count: const pick3 = random_chord_secret(Math.random, 3);

Source:
  • Mastermind.js, line 391
Example
random_chord_secret(Math.random, 3, ["C","D","E","F","G"])
// => e.g. ["E","C","G"]
Parameters:
Name Type Description
rng function

Zero-argument function returning a number in [0, 1). Pass Math.random for normal play.
count number

Number of notes to pick.
pool Array.<Note>

Notes to sample from (no duplicates expected).
Returns:

Array of count distinct notes.
